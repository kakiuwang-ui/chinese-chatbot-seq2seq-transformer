# 优化方案：解码策略与评测指标

> 本文档记录优化的**动机、问题、期望效果与方案**，对应实现见 `transformer.py` / `seq2seq.py` / `attn_seq2seq.py` 的解码部分与 `eval.py`。

---

## 1. 为什么要优化

三代模型已训练完成、技术演进对比清晰，但**回复的体感质量**仍有明显短板。值得注意的是：**这些短板大多不在模型本身，而在"解码策略"**——即训练好之后"怎么从概率分布里选词生成句子"。

> 关键认知：同一个训练好的模型，用不同解码策略，回复质量天差地别。优化解码**不需要重新训练**，性价比最高，且 Beam Search、采样等都是 NLP 核心考点，是答辩加分项。

同时，目前只有 ppl 一个指标，**缺少对"回复多样性"的量化**，无法用数据支撑"安全回复"问题的改善。

---

## 2. 目前有什么问题

| # | 问题 | 具体现象 | 成因 |
|---|---|---|---|
| P1 | **蹦 `<unk>`** | "wc"→`<unk>`、"这个效果怎么样"→`<unk>` | 低频词被映射为 `<unk>`，且训练目标含 `<unk>`，greedy 直接选了它 |
| P2 | **重复 / 复读** | "下楼下楼下楼下楼"、"麻哥他偏头痛，麻哥他偏头痛" | greedy 每步只挑当前最大概率，容易陷入重复循环 |
| P3 | **安全回复 / 单调** | 老回"对对""你的"等高频通用句 | greedy 倾向高频保守词；且数据一问多答，模型蒙通用答案 |
| P4 | **无多样性指标** | 只有 ppl，说不清回复"丰富"还是"单调" | 缺 Distinct 类指标 |

---

## 3. 希望优化到什么效果

| 目标 | 衡量方式 |
|---|---|
| 不再输出 `<unk>` | 人工检查 + 解码时禁止 |
| 消除明显复读 | 人工检查 + 重复惩罚 |
| 回复更丰富、不千篇一律 | **Distinct-1 / Distinct-2 指标上升** |
| 回复整体更优（全局而非贪心） | Beam Search；人工对比 |
| 有数据支撑优化效果 | 优化前后指标对比表 |

> 注意：目标**不是**把闲聊质量做到产品级（那需要海量数据+大模型+RLHF），而是用**标准解码技术**把现有模型的潜力释放出来，并量化证明。

---

## 4. 优化方案

### 4.1 解码策略（不重训，改 `reply`）

| 手段 | 解决问题 | 原理 |
|---|---|---|
| **禁止 `<unk>`** | P1 | 解码时把 `<unk>` 的概率置 -inf，强制选次优词 |
| **重复惩罚** | P2 | 对已生成过的词，降低其再次被选中的概率（logit ÷ penalty） |
| **Beam Search** | P3 | 同时保留 top-k 条候选序列，选**整句**概率最高的，而非每步贪心 |
| **Top-k 采样 + 温度** | P3 | 从前 k 个高概率词里按概率随机采样，回复更丰富不呆板 |

### 4.2 评测指标（新增 `eval.py`）

| 指标 | 含义 | 无需参考答案 |
|---|---|---|
| **Distinct-1** | 不重复单字 / 总字数，衡量用词多样性 | ✅ |
| **Distinct-2** | 不重复二元词组 / 总二元组，衡量短语多样性 | ✅ |
| 平均回复长度 | 辅助观察是否退化成短句 | ✅ |

对三个模型、在同一组测试问句上统一计算，得**优化前后 / 模型间对比表**。

---

## 5. 实测对比（10 句测试集，`python eval.py`）

| 模型 / 策略 | Distinct-1 | Distinct-2 | 平均长度 | 含 `<unk>` | 复读 |
|---|---|---|---|---|---|
| ① Seq2Seq (greedy+优化) | 0.671 | 0.947 | 8.5 | 无 | 无 |
| ② Attn-Seq2Seq (greedy+优化) | 0.667 | 0.954 | 7.5 | 无 | 无 |
| ③ Transformer (greedy) | **0.739** | 0.949 | 8.8 | 无 | 无 |
| ③ Transformer (beam) | 0.726 | **0.971** | **11.3** | 无 | 无 |
| ③ Transformer (topk) | 0.634 | 0.958 | 8.2 | 无 | 无 |

**观察**：
- 三模型均**已无 `<unk>`、无复读**（禁unk + 重复惩罚生效）。
- **Transformer 多样性最高**（Distinct-1 0.739），优于两代 RNN 模型。
- **Beam Search 回复最长最完整**（平均 11.3 字、Distinct-2 最高 0.971），印证"全局最优 > 逐步贪心"。
- Top-k 采样 Distinct-1 略低但回复更"活"（随机性带来口语化表达），适合追求趣味性的场景。
