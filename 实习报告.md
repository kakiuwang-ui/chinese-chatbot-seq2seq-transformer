# 基于 Seq2Seq 与 Transformer 的中文聊天机器人
## 《自然语言处理》综合实习报告

---

## 摘要

本实习以"中文闲聊聊天机器人"为载体，按 NLP 生成式模型的**真实演进顺序**，从零实现并对比三代模型：**Seq2Seq(LSTM) → Attention-Seq2Seq → Transformer**。不同于"直接调包跑一个最强模型"的做法，本项目刻意让每一代模型**暴露自身缺陷**，再用下一代技术去解决，从而完整覆盖词向量、RNN/LSTM、Encoder-Decoder、Attention、Transformer、文本生成等核心知识点。实验表明，三代模型在回复连贯度、困惑度、训练效率上呈现清晰的递进关系，验证了注意力机制与 Transformer 架构的有效性。

**关键词**：Seq2Seq、注意力机制、Transformer、文本生成、中文对话

## 1. 选题与目标

### 1.1 选题理由
课程提供六个题目，本组选择**题目二（聊天机器人）**，核心理由：

- 它是六题中**唯一覆盖完整 Transformer（Encoder-Decoder）** 的题目。文本生成任务必须实现 self-attention、masked self-attention、cross-attention、位置编码、自回归解码——而分类/匹配/序列标注类题目通常只用到 Encoder 半边，容易退化成"调包"。
- 与课程参考论文《Attention Is All You Need》直接对应，做这个题等于把论文逐模块复现。
- 技术路线 `词向量→Seq2Seq→Attention→Transformer` 是**递进式**的，便于体会"为什么需要下一个技术"。

### 1.2 设计目标
**深刻掌握**技术演进，而非追求产品级对话质量。因此评价标准是"**对比是否清晰、原理是否吃透**"，而非"是否像真人"。

## 2. 设计思路与方法论（思考过程记录）

### 2.1 核心方法：撞瓶颈式学习
本项目最关键的设计决策是：**不直接上最强模型，而是按历史顺序逐代实现，让每代模型亲自撞上自己的瓶颈。**

```
词向量 → LSTM → Seq2Seq ──❶固定context瓶颈──→ Attention ──❷依赖RNN不能并行──→ Transformer
```

- 先做 Seq2Seq 基线，**亲眼看到**长句回复崩坏 → 理解"固定 context 向量瓶颈"；
- 再加 Attention，**亲眼看到**长句改善、热力图对齐 → 理解注意力的作用；
- 最后做 Transformer，**亲眼看到**训练并行加速、连贯度提升 → 理解"为什么 Attention is all you need"。

这样掌握的是技术的**动机**，而非孤立的 API。

### 2.2 统一实验设置（保证公平对比）
三个模型共用同一套数据管线、同样的样本量(5万对)、同样的训练轮数(12 epoch)、相近的隐藏维度(256)，使 ppl 与回复质量的差异**只来自架构本身**。

### 2.3 工程组织
| 模块 | 文件 | 职责 |
|---|---|---|
| 数据管线 | `data_pipeline.py` | 解析、清洗、词表、DataLoader（三模型复用） |
| 阶段1 | `seq2seq.py` | Seq2Seq(LSTM) 基线 |
| 阶段2 | `attn_seq2seq.py` | Attention-Seq2Seq + 热力图 |
| 阶段3 | `transformer.py` | 从零手写 Transformer + 热力图 |
| 阶段4 | `compare.py` / `app.py` / `compare_chat.py` | 对比图、网页界面、终端对比 |

## 3. 数据与预处理

- **数据集**：小黄鸡 50w 中文闲聊语料（`xiaohuangji50w_nofenci.conv`）。
- **解析**：按 `E`/`M` 标记切出 (问, 答) 句对。
- **清洗**（针对小黄鸡噪声多、表情多的特点）：
  - 去除 emoji / 颜文字 / 特殊符号（正则只保留中英文、数字、常见标点）；
  - 过滤纯符号句、空句、过长句（词数 > 30）。
- **分词**：jieba 词级别分词。
- **词表**：`min_freq=2`，含 `<pad> <bos> <eos> <unk>` 四个特殊符。
- **结果**：5万样本清洗后得约 **45,360** 对，词表 **13,854**。

> 设计要点：目标序列包 `<bos>...<eos>`（解码器靠 `<bos>` 启动、靠 `<eos>` 停止）；loss 用 `ignore_index=<pad>` 屏蔽填充位。

## 4. 模型实现

### 4.1 阶段1：Seq2Seq (LSTM)
- **Encoder**：LSTM 将问句压成最终 `(hidden, cell)` —— 一个**固定长度**的 context 向量。
- **Decoder**：LSTM + 线性层，从 `<bos>` 起逐字生成，训练用 **Teacher forcing**（50% 概率喂真值）。
- **缺陷（预期）**：不论问句多长，都被压成一个 256 维向量 → **固定 context 瓶颈**。

### 4.2 阶段2：Attention-Seq2Seq
- Encoder 保留**每个时刻**的输出（不再只给最后一个向量）。
- 手写 **Bahdanau 加性注意力**：解码每个字时，对所有输入位置算 `tanh(W[s;h])→v→softmax` 得权重，加权求和得**动态 context**。
- **改善**：长句信息不再被压丢；注意力权重可导出**热力图**，直观显示"生成某字时在看输入哪些词"。

### 4.3 阶段3：Transformer（从零手写）
对照《Attention Is All You Need》逐模块实现：

| 模块 | 论文节 | 实现要点 |
|---|---|---|
| Scaled Dot-Product Attention | 3.2.1 | `QKᵀ/√d_k → mask → softmax → ·V` |
| Multi-Head Attention | 3.2.2 | 拆 8 头并行，再 concat |
| Positional Encoding | 3.5 | 正弦位置编码注入词序信息 |
| Masked self-attention | 3.2.3 | 上三角 look-ahead mask 防偷看未来 |
| Cross-attention | 3.2.3 | Q 来自解码器，K/V 来自编码器 |
| FFN/残差/LayerNorm | 3.3 | pre-LN 结构，小数据更稳 |

**三种 mask（核心难点）**：
1. **src-padding mask**：编码器自注意力 + 交叉注意力屏蔽输入 `<pad>`；
2. **look-ahead mask**：解码器自注意力屏蔽未来字（上三角）；
3. **tgt-padding mask**：解码器屏蔽目标 `<pad>`，与 2) 取交集。

**关键优势——并行训练**：靠 mask + teacher forcing，整句一次性算完，不像 LSTM 那样逐时间步循环。这是 Transformer 能扩展到大模型的根本原因。

## 5. 实验与结果

### 5.1 训练困惑度对比（统一：5万样本 / 12 epoch / Apple mps）

| 模型 | 起始 ppl | 最终 ppl | 收敛速度 | 训练速度 |
|---|---|---|---|---|
| Seq2Seq (LSTM) | 296.3 | **20.4** | 中 | 中 |
| Attention-Seq2Seq | 281.2 | 【待补】 | 更快(同期ppl更低) | 慢(逐步算注意力) |
| Transformer | 【待补】 | 【待补】 | 快 | **快(并行)** |

> 注：阶段2 前4个 epoch 的 ppl 为 281.2 / 153.2 / 92.9 / 59.3，同期已显著低于基线(113.5)，说明注意力加速了收敛。完整曲线见 `compare_ppl.png`。

### 5.2 对话质量对比（同一输入）

| 输入 | Seq2Seq | Transformer |
|---|---|---|
| 你好 | 我是\<unk\> | **你好可爱** |
| 真的吗 | 对，对， | **当然啦，我是你的** |
| 你这么会说话吗 | 你懂 | **我是小通** |
| （长句安慰请求） | 回复崩坏、重复 | 更连贯切题 |

**结论**：Transformer 回复**连贯、切题、有人格**，明显优于基线的"安全回复"。

## 6. 问题分析

### 6.1 安全回复(safe response)
基线常回"对对""你的"等高频通用句。原因：闲聊数据**一问多答**，模型为降 loss 学会蒙最高频回答。Attention/Transformer 缓解但未根除（属生成式对话固有难题）。

### 6.2 固定 context 瓶颈
Seq2Seq 长句崩坏的直接证据，是引入注意力的动机。

### 6.3 偶发 `<unk>`
低频/未登录词被映射为 `<unk>`，且训练目标中含 `<unk>`，greedy 解码偶尔输出。属小词表 + 小数据的固有局限。

## 7. 优化：解码策略与多样性提升（已实现）

> 详细动机/问题/方案见 `优化方案.md`。核心认知：**回复质量短板多在"解码策略"而非模型本身，优化解码无需重训。**

### 7.1 已实现的优化
| 手段 | 解决问题 | 实现 |
|---|---|---|
| 禁止生成 `<unk>` | 蹦 unk | 解码时把 `<unk>` logit 置 -inf |
| 重复惩罚 | 复读 | 已生成词的 logit ÷ 1.3 |
| Beam Search | 贪心局部最优 | 保留 top-3 候选序列，选整句最优（长度归一） |
| Top-k 采样 + 温度 | 回复单调 | 前 k 个高概率词按概率随机采样 |
| Distinct-1/2 评测 | 缺多样性指标 | `eval.py` 字级别 n-gram 多样性 |

### 7.2 优化效果（`eval.py`，10 句测试集）
| 模型 / 策略 | Distinct-1 | Distinct-2 | 平均长度 | `<unk>` | 复读 |
|---|---|---|---|---|---|
| ① Seq2Seq (greedy+优化) | 0.671 | 0.947 | 8.5 | 无 | 无 |
| ② Attn-Seq2Seq (greedy+优化) | 0.667 | 0.954 | 7.5 | 无 | 无 |
| ③ Transformer (greedy) | **0.739** | 0.949 | 8.8 | 无 | 无 |
| ③ Transformer (beam) | 0.726 | **0.971** | **11.3** | 无 | 无 |
| ③ Transformer (topk) | 0.634 | 0.958 | 8.2 | 无 | 无 |

**结论**：禁unk+重复惩罚消除了 `<unk>` 与复读；Transformer 多样性最高；Beam Search 回复最完整，印证"全局最优优于逐步贪心"。

### 7.3 仍可继续的方向（未实现）
| 手段 | 作用 | 需重训 |
|---|---|---|
| Noam warmup 学习率、Label smoothing | 还原论文训练技巧 | 是 |
| 数据加量、字级别词表、加大模型 | 治本但耗时 | 是 |
| BLEU（需参考答案） | 与标准答案接近度 | 否 |

## 8. 结论

本实习通过**从零实现三代生成式对话模型**，完整覆盖并深刻理解了词向量、LSTM、Seq2Seq、注意力机制、Transformer 等核心知识点。三代模型在困惑度、对话连贯度、训练效率上的清晰递进，实证了：

1. 注意力机制有效解决了 Seq2Seq 的**固定 context 瓶颈**；
2. Transformer 通过**纯注意力 + 并行训练**进一步提升质量与效率，印证"Attention is all you need"。

项目达成了"**深刻掌握技术演进**"的核心目标。
