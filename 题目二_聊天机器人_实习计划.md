## 二、分阶段计划

### 阶段 0：数据与基建
- **数据**：先用小黄鸡语料（小、干净），LCCC 留作后期加量。
- **预处理**：jieba 分词 → 建词表（含 `<pad> <bos> <eos> <unk>`）→ 文本转 id → padding。
- **基建**：`Dataset/DataLoader`、训练循环、loss 记录、`greedy_decode` 推理函数。
- **要搞懂**：为何需要 `<bos>/<eos>`？padding 处 loss 为何要 mask（`ignore_index`）？

### 阶段 1：Seq2Seq (LSTM)，不带 Attention
- **结构**：Encoder LSTM 压成固定 context 向量 → Decoder LSTM 逐字生成。
- **必学**：Teacher forcing、自回归解码、hidden state 传递。
- **关键体验**：试长句 → 回复变差 → 亲手撞上"固定 context 信息瓶颈"。
- **论文**：Sutskever 2014 Seq2Seq、Cho 2014 Encoder-Decoder。

### 阶段 2：Attention-Seq2Seq
- **改动**：Decoder 每步对 Encoder 所有时刻算注意力权重 → 动态 context。
- **自己手写**（勿调库）：Bahdanau（加性）或 Luong（点积）打分 → softmax → 加权。
- **关键体验**：存下 attention 权重画热力图，直观看到"生成某字时在看哪些输入词"。
- **论文**：Bahdanau 2015。

### 阶段 3：Transformer（重头戏，从零手写）
对照《Attention Is All You Need》逐节实现：

| 模块 | 论文节 | 易踩坑 |
|---|---|---|
| Scaled Dot-Product Attention | 3.2.1 | 除 √d_k；mask 用 -inf 再 softmax |
| Multi-Head Attention | 3.2.2 | 多头 reshape 维度顺序；最后 concat+线性 |
| Positional Encoding | 3.5 | 没有它无视词序；正弦/可学习都试 |
| Masked self-attention（解码器） | 3.2.3 | 上三角 mask 防看未来——最易错 |
| Cross-attention | 3.2.3 | Q 来自解码器，K/V 来自编码器，别接反 |
| FFN/残差/LayerNorm/warmup | 3.3, 5.3 | pre-LN vs post-LN 影响收敛 |

- **三种 mask 分清**：padding mask、look-ahead mask、cross-attention padding mask（最大混淆点，搞清就通 80%）。
- **关键体验**：同数据对比阶段 1/2/3 的 loss 与质量；解释 Transformer 为何能并行训练而 LSTM 不能。

### 阶段 4：整合、可视化、答辩
- **可视化（题目要求）**：对话流程图、Attention 热力图（阶段 2/3 各一张）、三模型 Loss 对比图、聊天界面（Gradio/Streamlit）。
- **答辩**：每人 2 分钟。建议你讲 Transformer（自注意力 + 三种 mask + 为何 "Attention is all you need"）。
