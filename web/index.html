NLP 综合实习 · 题目二 · 学习站

从 Seq2Seq → Transformer

不直接调用最强模型，而是按技术演进顺序逐代手写，让每一代亲自撞上自己的瓶颈——这样掌握的是「为什么」，而不是 API。

3 代模型 1 套数据管线 0 调包（核心模块全手写） 4.5w 中文对话对

方法论 · 为什么逐代手写

撞瓶颈式学习

深度学习最容易「会用不会懂」：调个预训练模型就能跑分类、跑生成，但说不清它为什么这么设计。本项目刻意反着来。

我们按真实历史顺序，从最简单的 Seq2Seq 开始，每实现一代就故意暴露它的缺陷——用长句把基线问崩，亲眼看到「固定 context 向量」装不下整句信息；再引入注意力去补这个洞；当注意力又受限于 RNN 无法并行时，才迎来 Transformer。每一步的动机都是上一步的痛点，技术不再是孤立的名词，而是一条有因果的链。

核心信念：理解一个技术的最佳方式，是先理解「没有它会怎样」。所以三代模型共用同一套数据、同样的训练规模，让差异只来自架构本身——这样的对比才有说服力。

阶段 0 · 一切的地基

数据从哪来，怎么洗

50w小黄鸡原始对话
4.5w清洗后句对（抽样5w）
13854词表大小
4特殊符号

数据是小黄鸡 50w 中文闲聊语料。这份数据真实但「脏」——大量颜文字、表情、纯符号、超短句。直接拿来训练会让模型学一身坏毛病，所以预处理这步本身就是工程亮点：

  1. 解析：按 E / M 标记，把原始文件切成「问 → 答」句对。
  2. 清洗：正则只保留中英文、数字与常见标点，删掉 emoji / 颜文字 / 特殊符号；过滤纯符号句、空句、词数超 30 的超长句。
  3. 分词：用 jieba 做词级别切分。
  4. 建词表：词频 ≥ 2 才入表，并预留 4 个特殊符号 —— <pad> 填充、<bos> 起始、<eos> 结束、<unk> 未登录词。
  5. 张量化：句子转 id，按 batch 动态 padding；目标序列包成 <bos> … <eos>。
两个易忽略的设计：① 目标必须带 <bos>/<eos> —— 解码器靠 <bos> 启动、靠预测出 <eos> 知道何时停笔。② 计算损失时用 ignore_index=<pad> 把填充位屏蔽，否则模型会把大量精力浪费在「预测 pad」上。

技术演进 · 一条解决缺陷的链条

每一代，都为修补上一代而生

这五步不是并列的知识点，而是一条因果链。读懂「下一个为什么出现」，整条链就通了。

  1. 01

    词向量 · Word2Vec

    把字词变成稠密向量，机器才能计算。one-hot 维度爆炸且词与词之间正交（无相似度），词向量用低维连续空间让「猫」和「狗」彼此靠近。

    nn.Embedding
  2. 02

    RNN / LSTM

    让模型按时间步处理变长序列、携带「记忆」。RNN 会梯度消失、记不住长依赖；LSTM 用遗忘门 / 输入门 / 输出门控制信息的保留与丢弃，能记更久。

    hidden / cell state
  3. 03

    Seq2Seq · 编码器-解码器

    两个 LSTM：编码器读问句、压成一个向量，解码器据此逐字生成答句。开创了「序列进、序列出」的范式。缺陷：不论问句多长，都被压成一个固定向量，长句信息严重丢失。

    Teacher forcing
  4. 04

    Attention · 注意力机制

    解码每个字时，对编码器所有时刻算一组权重、动态加权求和，得到「这一刻该看哪里」的 context。彻底解决固定向量瓶颈。缺陷：注意力仍架在 RNN 之上，必须按时间步串行，无法并行、长依赖仍偏弱。

    动态 context
  5. 05

    Transformer · Attention is all you need

    干脆扔掉 RNN，用纯注意力堆叠 + 位置编码搭建编码器/解码器。训练可整句并行，长依赖一步直达，是现代大模型的共同基石。

    multi-head · 3 种 mask · 并行

阶段 1 · 从零手写 · 对照 Sutskever 2014

Seq2Seq 是怎么实现的

最朴素的一代：两个 LSTM，一个读、一个写。不带任何注意力——正因如此，它会亲手撞上「固定 context 瓶颈」，这个痛点就是下一节引入注意力的全部动机。下面每一块都配上项目里的真实代码。

统一配置（与后两代对齐，只让架构变量单独发挥）： emb_dim=128 · hid_dim=256 · num_layers=1 · dropout=0.1 · teacher_forcing=0.5 · 5 万样本 · Apple MPS。整体数据流：问句 → Encoder-LSTM 压成 (hidden, cell) → Decoder-LSTM 从 <bos> 逐字生成 → 线性投影到词表。训练 12 轮，Adam lr=1e-3 + 梯度裁剪，ppl 约 9.9。
模块 01 · 瓶颈的源头

Encoder：把整句压成一个向量

怎么做：词向量过一层 LSTM，只保留最后一步的 (hidden, cell) 作为整句的全部「记忆」，交给解码器——注意它丢弃了所有中间时刻的输出。

为什么要 pack：一个 batch 里短句被补了 <pad>，若不处理，LSTM 会把 pad 也算进最后一步，污染 context。pack_padded_sequence 让 LSTM 跳过 pad，取到每句真实的最后一步。

瓶颈在哪：不论问句 3 个字还是 30 个字，都被压进同一个固定长度向量——句子越长，信息丢得越多，这是本代的死穴。

模块 02 · 一次只走一步

Decoder：吃「上一个字 + 上一步状态」

怎么做：解码器是单步的：每次输入一个字 + 上一步的 (hidden, cell)，LSTM 走一步，再用一层 Linear 投影到整个词表，得到下一个字的分布。

为什么单步：生成是自回归的——下一个字依赖已生成的字，无法一次算完。这也正是 RNN 系「必须串行、不能并行」的根源，直到 Transformer 才被打破。

模块 03 · 训练的关键技巧

Seq2Seq 主循环：teacher forcing

怎么做：Encoder 给出 context 后，从 <bos> 起逐时间步解码。每一步以 0.5 的概率把下一步输入换成「真值上一字」（teacher forcing），否则喂模型自己的预测。

为什么要它：训练早期模型预测很差，若一直喂自己的错误输出，误差会滚雪球、迟迟学不动。喂真值能让训练稳定、收敛快。

代价：训练总喂真值、推理只能喂自己的预测，这个差异叫 exposure bias——训练与推理的口径不一致。

模块 04 · 对齐与稳定

训练：错位对齐 + 屏蔽 pad + 裁剪

怎么做：算损失前把输出与目标错开一位——丢掉第 0 位的 <bos>，让「预测」和「真值下一字」对齐。CrossEntropyLoss(ignore_index=<pad>) 让填充位不计入损失。

为什么裁剪：LSTM 在长序列上容易梯度爆炸，clip_grad_norm_ 把梯度范数限制在 1.0，是 RNN 训练的常规保险。

模块 05 · 自回归生成

推理：从 <bos> 贪心解码到 <eos>

怎么做：推理没有真值，只能自回归：从 <bos> 起，每步取概率最大的字（greedy）喂回下一步，直到吐出 <eos> 或达到长度上限。

两个不重训的小优化：把 <unk> 的 logit 直接压到 −∞（永不输出未登录词）；对已生成过的字做重复惩罚，打断「对对对」式复读。这两招后两代也共用。

撞上瓶颈（这正是全项目的起点）：短句还行，但一旦问句变长，回复就明显跑偏、变「安全套话」——因为整句被硬压进一个固定向量，长程信息在编码时就已丢失。这不是 bug，而是预期：只有亲眼看到它崩，才真正理解下一节注意力机制为什么必须出现——让解码每一步都能回头动态地看输入，而不再依赖那一个被压扁的向量。

深入 · 整个 Transformer 的心脏

注意力到底在算什么

一句话：用「查询」去和一堆「键」比相似度，再据此对「值」加权求和。把它想成一次软性的查字典。

Q · Query

当前这个字的「我想找什么」。

K · Key

每个位置的「我是什么」，供 Q 来匹配。

V · Value

每个位置真正携带的信息，按匹配度被取用。

先用「查字典 / 检索」建立直觉：要读懂「小猫喝牛奶，它很满足」里的「它」到底指谁——Query 是「它」举手提问「我在找前文一个能被『满足』的东西」；句中每个词都挂着一块 Key 招牌写着「我是谁」（"小猫"的招牌：名词·动物·可作主语）；Value 则是这个词真正携带的语义内容。拿 Query 挨个去比对每块 Key 的相似度，越匹配分越高；把分数 softmax 归一成权重；再按权重把各词的 Value 混合起来——「它」于是自动「看向」了「小猫」。提问 → 匹配 → 取值，一次注意力就是一次软性的查字典，而且不需要像 RNN 那样一步步爬过去，任意两个词一步直连。
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V Q·Kᵀ 算相似度 → softmax 归一成权重 → 对 V 加权求和
为什么要除以 √d_k？ 向量维度越高，点积的数值越容易变得很大，把 softmax 推进「几乎一个位置占满」的饱和区，梯度趋近 0、难以训练。除以 √d_k 把方差拉回 ~1，让分布平滑、梯度健康。
为什么要「多头」？ 一组 Q/K/V 只能学一种关系。多头把向量拆成 h 份（本项目 8 头），各自在不同子空间并行注意，再拼接——有的头管语法、有的头管指代，模型同时捕捉多种关系。

深入 · 初学者最大的混淆点

三种 mask，搞清就通了八成

mask 的作用很统一：把不该看的位置在 softmax 之前置成 −∞，让它的权重变成 0。区别只在「屏蔽谁」。

① src-padding mask

屏蔽输入的 <pad>

编码器自注意力 + 解码器交叉注意力中使用。一个 batch 里短句被补了 pad，不能让注意力把这些无意义的填充也算进去。

② look-ahead mask

屏蔽「未来的字」

解码器自注意力专用，是个上三角矩阵。生成第 t 个字时只能看 1…t，不能偷看 t 之后——否则训练时等于抄答案，推理时就废了。

③ tgt-padding mask

屏蔽目标的 <pad>

解码器里和 ② 取交集。既不能看未来，也不能看目标序列里的填充位，两个条件同时满足。

关键实现细节：三种 mask 都是先把被屏蔽位置的注意力分数 masked_fill(mask==0, -1e9)，再过 softmax —— 因为 e−∞ ≈ 0，这些位置自然分不到权重。look-ahead mask 用 torch.tril（下三角为 1）一行生成。

深入 · 另外两个关键设计

位置编码，与「为什么能并行」

位置编码 · Positional Encoding

注意力本身是对位置无感的——打乱输入顺序，输出不变。但「我打你」和「你打我」意思天差地别，词序必须保留。Transformer 没有 RNN 的天然时序，于是把位置信息显式注入词向量。

PE(pos, 2i)  = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d) 不同维度用不同频率的正弦/余弦，组合出每个位置独一无二的编码

为什么 Transformer 能并行训练

这是它取代 RNN 的根本原因，也是答辩必讲点。

  1. RNN 系（Seq2Seq / Attn）：解码必须一个时间步算完再算下一步，是 Python 里的串行循环，慢。
  2. Transformer：训练时整个目标句一次性喂入，靠 look-ahead mask 保证每个位置只看得到自己之前的字。
  3. 结果：所有位置的损失在一次前向里并行算完，无时间步循环——这正是它能扩展到超大模型的关键。
Teacher forcing 是什么？ 训练时，解码器每一步的输入喂的是「真值的上一个字」而非自己的预测，让训练稳定、收敛快。推理时没有真值，只能喂自己上一步的输出（自回归）——训练与推理的这点差异称为 exposure bias。

阶段 3 · 从零手写 · 对照《Attention Is All You Need》

Transformer 是怎么实现的

不调 nn.Transformer，逐模块对照原论文手写：多头注意力、三种 mask、位置编码、Pre-LN 残差块，再到并行训练与解码优化。下面每一块都配上项目里的真实代码。

统一配置（与前两代对齐，只让架构变量单独发挥）： d_model=256 · n_heads=8 · n_layers=3 · d_ff=512 · dropout=0.1 · 5 万样本 · Apple MPS。整体数据流：Embedding×√d_model → 位置编码 → 3 层 Encoder → 3 层 Decoder → 线性投影到词表。训练用 Noam warmup（2000 步预热）跑 20 轮——见文末结果区，正是它把 ppl 从 12.9 压到 4.48。
模块 01 · 心脏

多头注意力（含 Scaled Dot-Product）

怎么做：三个线性层把输入投影成 Q/K/V，用 view+transpose 拆成 8 个头（[B,len,256]→[B,8,len,32]）；核心就一行 QKᵀ/√d_k，softmax 后加权 V，再合并回来过 wo。

为什么除以 √d_k：维度越高点积越大，会把 softmax 推入饱和区、梯度趋 0。除以 √d_k（这里 =√32≈5.66）把方差拉回 ~1。

为什么多头：单组 Q/K/V 只能学一种关系；8 个头在不同子空间并行注意，有的管语法、有的管指代。self.attn 存下权重，专为画交叉注意力热力图。

模块 02 · 初学者最大混淆点

三种 mask，两个函数搞定

怎么做：make_src_mask 只做 padding mask；make_tgt_mask 把「目标 pad」和 torch.tril 上三角 取交集（pad & sub），一步同时满足「不看 pad」＋「不看未来」。

为什么要上三角：训练时整句一起喂进去，若不挡未来，预测第 t 字时就能看到第 t 字本身＝抄答案，推理必崩。它同时也是能并行训练的前提。

屏蔽统一用 masked_fill(mask==0,-1e9)，因 e−∞≈0，softmax 后权重自然归零。

模块 03 · 残差块

Decoder 层：Pre-LN 的三级子层

怎么做：每个子层前先 LayerNorm（Pre-LN），再进注意力/前馈，最后残差相加。解码器有三级：①掩码自注意力 → ②交叉注意力（Q 来自解码器、K/V 来自编码器）→ ③前馈。

为什么用 Pre-LN 而非论文的 Post-LN：小数据、无 warmup 时 Post-LN 容易训崩；Pre-LN 梯度更平滑、更稳，代价是最终精度略低——一个务实取舍。

模块 04 · 弥补位置无感

正弦位置编码

怎么做：偶数维用 sin、奇数维用 cos，不同维度用不同频率，组合出每个位置独一无二的编码；register_buffer 存成不参与训练的常量。

为什么需要它：注意力对顺序无感——「我打你」和「你打我」会被当成一样。Transformer 没有 RNN 的天然时序，必须把位置显式注入词向量。

Embedding 还要 ×√d_model（对齐论文 3.4），让语义量级和位置编码匹配，不被淹没。

模块 05 · 取代 RNN 的根本原因

为什么能整句并行训练

怎么做：解码器输入取 tgt[:, :-1]、目标取 tgt[:, 1:]（右移一位），一次前向就算完整句每个位置的损失，没有 LSTM 那样的时间步循环。

为什么这是关键：RNN 系解码必须一步算完再算下一步（Python 串行循环，慢）；Transformer 靠 look-ahead mask 保证每个位置只看自己之前的字，于是所有位置并行——这正是它能扩展到超大模型的原因。

模块 06 · 不重训也能提升

解码优化：禁 <unk> + 重复惩罚

怎么做：解码每步把 <unk> 的 logit 直接置 −∞（永不输出未登录词）；对已生成过的字，logit 除以惩罚系数 1.3 压低，打断「下楼下楼下楼」式复读。

为什么有用：两招都只改解码、无需重训，配合 greedy / beam（长度归一）/ top-k 三种策略，回复质量天差地别。

效果（实测，与前两代同数据同规模）：Transformer 训练最快（整句并行），真实生成的回复最长最连贯（均长 11–12，三代最高）。曾经反直觉的是它 12 轮时 ppl=12.9 反而最高——我们判断是欠拟合 + 缺学习率预热，遂加 Noam warmup 训到 20 轮，ppl 降到 4.48，反超成为三代最低。这印证了架构本身没问题；但也别忘了 ppl 只衡量 teacher forcing 下的下一字预测，不等于生成质量——这仍是本项目最重要的发现之一。

实验现象 · 撞瓶颈式学习

同一句话，三代回复的落差

基线的「安全回复」和长句崩坏不是 bug，而是引入注意力与 Transformer 的动机。

两个固有难题：① 安全回复——闲聊数据「一问多答」，模型为降低损失学会蒙「对对」「你的」这类高频通用句；② 固定 context 瓶颈——基线长句直接崩坏。注意力与 Transformer 缓解了它们，但生成式对话本身难以根治，这是诚实的边界。

从「反直觉」到闭环——本项目最重要的一课：同为 12 轮时，Transformer 的 ppl=12.9 反而最高，很反直觉。 我们没有归咎于架构，而是判断它容量更大、又缺学习率预热，属于欠拟合。于是给它加上 Noam warmup 并训到 20 轮， ppl 从 12.9 骤降到 4.48，反超 Attn 成为三代最低，回复也更长更连贯——证明架构没问题，是训练配置的锅。 但即便如此，仍要记住：ppl 只衡量 teacher forcing 下的下一字预测，不等于真实自回归生成质量（看 Distinct：D-2 最高的其实是 Attn）。 评价生成模型，永远不能只看一个指标。

深入 · 复现论文模型的隐形门槛

原先怎么训的，为什么加 warmup 会好

同一套架构、同一份数据，只改「学习率随训练怎么走」，困惑度就从 12.9 掉到 4.48。

原先 · 固定学习率

lr ≡ 5e-4，训 12 轮

Adam 全程恒定学习率，没有任何预热。图省事，但对大模型不友好——ppl 卡在 12.9，是三代里最高的。

现在 · Noam warmup

先升温、再衰减，训 20 轮

前 2000 步学习率线性爬升，之后按 step−0.5 缓降，峰值落在第 2000 步。ppl → 4.48，反超 Attn。

差别 · 只动了调度

架构 / 数据 / 规模全不变

没加参数、没换结构、没加数据。唯一变量是「学习率随步数怎么变」——这就是训练技巧的分量。

lr = d_model−0.5 · min( step−0.5,  step · warmup−1.5 ) step < warmup：lr ∝ step（线性升温）　·　step > warmup：lr ∝ step−0.5（缓慢衰减）　·　峰值在 step = warmup
代码 · 到底改了什么

两行调度，替掉一个常数

原先：建好 Adam 就把 lr=5e-4 定死，训练循环里再不碰它。

现在：每个 step 先按 Noam 公式算出当前 lr、写回优化器，再 step()。warmup=2000，训练拉到 20 轮。

注意 Adam 的 betas=(0.9, 0.98) 也对齐论文——β₂ 从常见的 0.999 调到 0.98，让方差估计更快跟上，与 warmup 配套。

为什么它真的有效（三条机理）

  1. 给 Adam 时间把「方差估计」稳住。Adam 的自适应步长 ≈ lr / √v，其中 v 是梯度平方的滑动平均。训练最初几十步 v 的样本太少、噪声极大，直接上大 lr 会让步长忽大忽小，把刚初始化的权重甩飞。先用小 lr 预热，等 v 估计稳定了再提速。
  2. 别在起跑就把注意力推进饱和区。刚初始化时若 lr 过大，softmax(QKᵀ/√d_k) 很容易被推到「一个位置吃满」的饱和区，梯度≈0 就再也学不动（这正是前面讲 √d_k 时同一个坑）。小 lr 起步，让注意力分布先平滑地长开。
  3. 大模型 + 小数据，固定 lr 顾此失彼。一个恒定 lr 要么初期太大（前期崩），要么后期太小（收敛慢）。warmup「先小 → 到峰值 → 再缓降」正好兼顾初期稳定与中期快速收敛——这也是它几乎成为复现 Transformer 标配的原因。

怎么读这张对比：warmup 最确定的收益是训练收敛——ppl 从 12.9 近乎腰斩到 4.48；生成层面回复明显变长（更敢展开），多样性 Distinct 基本持平、个别略降。也就是说，它主要修好的是「训练」这一环，而非直接改写生成风格——这也是它诚实的边界。

总结：warmup 不改变模型「能表达什么」，只改变优化器怎么走到那里。对 RNN/LSTM 这类小模型它可有可无，但对 Transformer——大模型、Adam、注意力饱和风险叠加——它几乎是「能不能训出来」的分水岭。这正是我们从 ppl=12.9 的教训里学到的：复现论文级模型，必须配套论文级的训练技巧。

优化 · 不重训也能提升的部分

训练好之后，怎么「选词」也很关键

同一个模型，换一种解码策略，回复质量天差地别。三者皆已加：禁 <unk> + 重复惩罚。

greedy

你好可爱

每步只挑当前概率最大的字。最快，但容易陷入重复，并偏好高频的「安全回复」。

beam search

为什么要幸福，开心点嘛？

同时保留 top-k 条候选序列，最终选整句概率最高的，而非每步贪心。回复最完整，Distinct-2 最高。

top-k 采样

是你才怪呢你全家都好

在前 k 个高概率字里按概率随机采样，引入可控随机性。回复更「活」、口语化、不呆板。

禁 <unk> + 重复惩罚怎么做的：解码每步把 <unk> 的 logit 直接置 −∞（永不输出未登录词）；对已经生成过的字，把它的 logit 除以一个惩罚系数（1.3）来压低，打断「下楼下楼下楼」式的复读。两招都只改解码、无需重训。

附录 · 关键概念速查

一页看懂这些名词

词向量 Embedding
把离散的字词映射成低维稠密向量，相近语义彼此靠近。
Teacher forcing
训练时解码器输入喂真值上一字，加速且稳定收敛。
自回归解码
推理时逐字生成，每步喂自己上一步的输出。
固定 context 瓶颈
Seq2Seq 把整句压成一个向量，长句信息丢失。
Q / K / V
查询 / 键 / 值，注意力的三要素：用 Q 匹配 K，加权取 V。
Scaled Dot-Product
点积相似度除以 √d_k 再 softmax，稳定梯度。
Multi-Head
把注意力拆成多个子空间并行，捕捉多种关系。
位置编码
用正弦/余弦把词序显式注入，弥补注意力的位置无感。
look-ahead mask
上三角掩码，禁止解码器偷看未来的字。
困惑度 Perplexity
预测下一字不确定性的指标，越低越好（≠生成质量）。
Distinct-n
不重复 n-gram 占比，衡量回复多样性。
Beam Search
保留多条候选、选整句最优的解码策略。
学习率 lr
每步参数沿梯度反方向更新的步长。太大易震荡/发散，太小收敛慢。
Noam warmup
学习率先线性升温、后按 step⁻⁰·⁵ 衰减的调度，Transformer 收敛的关键。
