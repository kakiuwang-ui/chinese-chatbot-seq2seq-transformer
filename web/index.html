从 Seq2Seq → Transformer

不直接调用最强模型，而是按技术演进顺序逐代手写，让每一代亲自撞上自己的瓶颈——这样掌握的是「为什么」，而不是 API。

3 代模型 1 套数据管线 0 调包（核心模块全手写） 4.5w 中文对话对

方法论 · 为什么逐代手写

撞瓶颈式学习

深度学习最容易「会用不会懂」：调个预训练模型就能跑分类、跑生成，但说不清它为什么这么设计。本项目刻意反着来。

我们按真实历史顺序，从最简单的 Seq2Seq 开始，每实现一代就故意暴露它的缺陷——用长句把基线问崩，亲眼看到「固定 context 向量」装不下整句信息；再引入注意力去补这个洞；当注意力又受限于 RNN 无法并行时，才迎来 Transformer。每一步的动机都是上一步的痛点，技术不再是孤立的名词，而是一条有因果的链。

核心信念：理解一个技术的最佳方式，是先理解「没有它会怎样」。所以三代模型共用同一套数据、同样的训练规模，让差异只来自架构本身——这样的对比才有说服力。

阶段 0 · 一切的地基

数据从哪来，怎么洗

50w小黄鸡原始对话
4.5w清洗后句对（抽样5w）
13854词表大小
4特殊符号

数据是小黄鸡 50w 中文闲聊语料。这份数据真实但「脏」——大量颜文字、表情、纯符号、超短句。直接拿来训练会让模型学一身坏毛病，所以预处理这步本身就是工程亮点：

  1. 解析：按 E / M 标记，把原始文件切成「问 → 答」句对。
  2. 清洗：正则只保留中英文、数字与常见标点，删掉 emoji / 颜文字 / 特殊符号；过滤纯符号句、空句、词数超 30 的超长句。
  3. 分词：用 jieba 做词级别切分。
  4. 建词表：词频 ≥ 2 才入表，并预留 4 个特殊符号 —— <pad> 填充、<bos> 起始、<eos> 结束、<unk> 未登录词。
  5. 张量化：句子转 id，按 batch 动态 padding；目标序列包成 <bos> … <eos>。
两个易忽略的设计：① 目标必须带 <bos>/<eos> —— 解码器靠 <bos> 启动、靠预测出 <eos> 知道何时停笔。② 计算损失时用 ignore_index=<pad> 把填充位屏蔽，否则模型会把大量精力浪费在「预测 pad」上。

技术演进 · 一条解决缺陷的链条

每一代，都为修补上一代而生

这五步不是并列的知识点，而是一条因果链。读懂「下一个为什么出现」，整条链就通了。

  1. 01

    词向量 · Word2Vec

    把字词变成稠密向量，机器才能计算。one-hot 维度爆炸且词与词之间正交（无相似度），词向量用低维连续空间让「猫」和「狗」彼此靠近。

    nn.Embedding
  2. 02

    RNN / LSTM

    让模型按时间步处理变长序列、携带「记忆」。RNN 会梯度消失、记不住长依赖；LSTM 用遗忘门 / 输入门 / 输出门控制信息的保留与丢弃，能记更久。

    hidden / cell state
  3. 03

    Seq2Seq · 编码器-解码器

    两个 LSTM：编码器读问句、压成一个向量，解码器据此逐字生成答句。开创了「序列进、序列出」的范式。缺陷：不论问句多长，都被压成一个固定向量，长句信息严重丢失。

    Teacher forcing
  4. 04

    Attention · 注意力机制

    解码每个字时，对编码器所有时刻算一组权重、动态加权求和，得到「这一刻该看哪里」的 context。彻底解决固定向量瓶颈。缺陷：注意力仍架在 RNN 之上，必须按时间步串行，无法并行、长依赖仍偏弱。

    动态 context
  5. 05

    Transformer · Attention is all you need

    干脆扔掉 RNN，用纯注意力堆叠 + 位置编码搭建编码器/解码器。训练可整句并行，长依赖一步直达，是现代大模型的共同基石。

    multi-head · 3 种 mask · 并行

STAGE 02

Attention-Seq2Seq

编码器保留每个时刻的输出；解码时手写加性注意力，按需取用。

动态 context，逐字决定看输入哪里

# 手写 Bahdanau 加性打分

STAGE 03

Transformer

从零手写多头注意力、位置编码、三种 mask，对照原论文逐模块复现。

纯注意力，无 RNN，长依赖一步直达

整句并行训练，速度最快

生成多样性、连贯度最佳

# Scaled Dot-Product
scores = Q·Kᵀ / sqrt(d_k)
scores = scores.masked_fill(mask==0,-1e9)
attn = softmax(scores)
out  = attn · V

深入 · 整个 Transformer 的心脏

注意力到底在算什么

一句话：用「查询」去和一堆「键」比相似度，再据此对「值」加权求和。把它想成一次软性的查字典。

Q · Query

当前这个字的「我想找什么」。

K · Key

每个位置的「我是什么」，供 Q 来匹配。

V · Value

每个位置真正携带的信息，按匹配度被取用。

Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V Q·Kᵀ 算相似度 → softmax 归一成权重 → 对 V 加权求和
为什么要除以 √d_k？ 向量维度越高，点积的数值越容易变得很大，把 softmax 推进「几乎一个位置占满」的饱和区，梯度趋近 0、难以训练。除以 √d_k 把方差拉回 ~1，让分布平滑、梯度健康。
为什么要「多头」？ 一组 Q/K/V 只能学一种关系。多头把向量拆成 h 份（本项目 8 头），各自在不同子空间并行注意，再拼接——有的头管语法、有的头管指代，模型同时捕捉多种关系。

深入 · 初学者最大的混淆点

三种 mask，搞清就通了八成

mask 的作用很统一：把不该看的位置在 softmax 之前置成 −∞，让它的权重变成 0。区别只在「屏蔽谁」。

① src-padding mask

屏蔽输入的 <pad>

编码器自注意力 + 解码器交叉注意力中使用。一个 batch 里短句被补了 pad，不能让注意力把这些无意义的填充也算进去。

② look-ahead mask

屏蔽「未来的字」

解码器自注意力专用，是个上三角矩阵。生成第 t 个字时只能看 1…t，不能偷看 t 之后——否则训练时等于抄答案，推理时就废了。

③ tgt-padding mask

屏蔽目标的 <pad>

解码器里和 ② 取交集。既不能看未来，也不能看目标序列里的填充位，两个条件同时满足。

关键实现细节：三种 mask 都是先把被屏蔽位置的注意力分数 masked_fill(mask==0, -1e9)，再过 softmax —— 因为 e−∞ ≈ 0，这些位置自然分不到权重。look-ahead mask 用 torch.tril（下三角为 1）一行生成。

深入 · 另外两个关键设计

位置编码，与「为什么能并行」

位置编码 · Positional Encoding

注意力本身是对位置无感的——打乱输入顺序，输出不变。但「我打你」和「你打我」意思天差地别，词序必须保留。Transformer 没有 RNN 的天然时序，于是把位置信息显式注入词向量。

PE(pos, 2i)  = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d) 不同维度用不同频率的正弦/余弦，组合出每个位置独一无二的编码

为什么 Transformer 能并行训练

这是它取代 RNN 的根本原因，也是答辩必讲点。

  1. RNN 系（Seq2Seq / Attn）：解码必须一个时间步算完再算下一步，是 Python 里的串行循环，慢。
  2. Transformer：训练时整个目标句一次性喂入，靠 look-ahead mask 保证每个位置只看得到自己之前的字。
  3. 结果：所有位置的损失在一次前向里并行算完，无时间步循环——这正是它能扩展到超大模型的关键。
Teacher forcing 是什么？ 训练时，解码器每一步的输入喂的是「真值的上一个字」而非自己的预测，让训练稳定、收敛快。推理时没有真值，只能喂自己上一步的输出（自回归）——训练与推理的这点差异称为 exposure bias。

实验现象 · 撞瓶颈式学习

同一句话，三代回复的落差

基线的「安全回复」和长句崩坏不是 bug，而是引入注意力与 Transformer 的动机。

输入① Seq2Seq③ Transformer
你好我是 <unk>你好可爱
真的吗对，对，当然啦，我是你的
你这么会说话吗你懂我是小通
你今天心情怎么样（重复 / 跑偏）那就和我聊天了，我抱着你的吧
两个固有难题：① 安全回复——闲聊数据「一问多答」，模型为降低损失学会蒙「对对」「你的」这类高频通用句；② 固定 context 瓶颈——基线长句直接崩坏。注意力与 Transformer 缓解了它们，但生成式对话本身难以根治，这是诚实的边界。

为什么 Transformer 的 ppl 反而不是最低？ 困惑度衡量的是 teacher forcing 下「预测下一字」的准确度， 而非真实自回归生成的连贯度。Transformer 在真实生成（对话、Distinct 多样性）上最优；它 ppl 偏高， 源于仅训练 12 轮、未用 Noam warmup，相对其容量训练不足。评价生成模型不能只看一个指标——这是本项目最重要的发现之一。

优化 · 不重训也能提升的部分

训练好之后，怎么「选词」也很关键

同一个模型，换一种解码策略，回复质量天差地别。三者皆已加：禁 <unk> + 重复惩罚。

greedy

你好可爱

每步只挑当前概率最大的字。最快，但容易陷入重复，并偏好高频的「安全回复」。

beam search

为什么要幸福，开心点嘛？

同时保留 top-k 条候选序列，最终选整句概率最高的，而非每步贪心。回复最完整，Distinct-2 最高。

top-k 采样

是你才怪呢你全家都好

在前 k 个高概率字里按概率随机采样，引入可控随机性。回复更「活」、口语化、不呆板。

禁 <unk> + 重复惩罚怎么做的：解码每步把 <unk> 的 logit 直接置 −∞（永不输出未登录词）；对已经生成过的字，把它的 logit 除以一个惩罚系数（1.3）来压低，打断「下楼下楼下楼」式的复读。两招都只改解码、无需重训。
